## Overview

Milestone v1.4 extended Scriveno's trusted surface in two careful directions. It added Perplexity Desktop as a guided, truth-first runtime target, then introduced a first-pass technical-writing family that feels native inside Scriveno's adaptive model. The milestone finished by locking those new surfaces into docs, packaging, and regression coverage so the launch story stays aligned with the live repo.

## Milestone Summary

**Key Decisions:**

- Treat Perplexity Desktop as a guided local-MCP target instead of pretending it is a normal slash-command runtime
- Keep runtime truth centralized in `docs/runtime-support.md` and protected by tests
- Introduce technical writing as a dedicated family with a small, coherent first-pass scope
- Keep technical-writing exports narrow until a later milestone proves broader publishing behavior

**Issues Resolved:**

- Added a truthful Perplexity install path without weakening the v1.3 runtime-credibility posture
- Eliminated fiction-centric defaults for the first technical-writing work types
- Updated root, onboarding, and instruction docs to the new 50-work-type / 9-group truth surface
- Added packaging and regression checks so the new runtime and technical surfaces do not silently drift

**Issues Deferred:**

- Broader Perplexity support beyond the Desktop local-MCP surface
- Docs-site or portal-oriented publishing for technical-writing projects
- A larger technical-writing catalog beyond guides, runbooks, API references, and design specs

**Technical Debt Incurred:**

- None recorded for milestone scope
