# Requirements: Scriveno v1.8 Command Surface Coherence

**Status:** Shipped 2026-04-18.

**Goal:** Make Scriveno's command surface truthful and runtime-native so every documented command name is runnable in the host that advertises it.

**Research input:** Review findings from 2026-04-17 plus command-surface precedent from command-directory runtimes that expose flat host-native slash commands.

---

## v1.8 Requirements

### Canonical Command Surface

- [x] **CMD-01**: Sacred-exclusive commands expose one canonical installed runtime name per host surface, and all command files, docs, and generated manifests now reference that real installed name
- [x] **CMD-02**: Installer/runtime inventory generation now comes from one canonical command-source model so sacred commands no longer appear as both phantom top-level commands and namespaced commands in the same runtime
- [x] **CMD-03**: Cross-file validation now detects command references that do not resolve against the installed runtime surface for the target host

### Adapted Alias Truthfulness

- [x] **CMD-04**: Adapted command names shown to writers are now reframed as descriptive adaptations unless a runtime explicitly installs wrappers or aliases
- [x] **CMD-05**: Help, router, sacred docs, work-type docs, and command reference now stay aligned on adapted names as conceptual labels rather than silent command promises

### Claude Flat Command Surface

- [x] **CMD-06**: Claude Code now uses one flat `/scr-*` command surface consistently across installer output, docs, examples, and upgrade guidance
- [x] **CMD-07**: Claude-facing command examples for autopilot/autonomous-style flows now use the flat `/scr-*` surface consistently
- [x] **CMD-08**: Remaining Claude-specific `/scr:*` examples in launch, guide, or reference docs have been normalized or explicitly framed as non-Claude surfaces

### Regression and Contributor Guidance

- [x] **CMD-09**: Regression tests now cover nested sacred command discovery plus dead-reference detection across `commands/`, `docs/`, and installer-generated manifests
- [x] **CMD-10**: Contributor/runtime docs now clearly explain the naming contract by host: Claude uses `/scr-*`, Codex uses `$scr-*`, and namespaced or adapted forms are marked installed vs conceptual

---

## Out of Scope

- Rewriting command behavior itself beyond what was required to make the advertised invocation surface truthful
- Introducing a compiled command dispatcher or runtime dependency to solve naming indirection
- Broad new product features unrelated to command-surface consistency
- Cross-runtime parity claims beyond the install surfaces Scriveno can actually verify
