    Create `commands/scr/cleanup.md` following Pattern 1 (structural invariant) exactly:

    **Frontmatter** (adapt from health.md):
    ```
    description: Strip template scaffold markers from draft files. Dry-run by default.
    argument-hint: "[--apply]"
    ```

    **H1 heading:** `# /scr:cleanup -- Scaffold Cleanup`

    **Brief description paragraph:** One short paragraph explaining what the command does (strip bracket markers, Alternate blocks, and duplicate H1 headings from `.manuscript/drafts/`).

    **## Usage block** with flags:
    - `--apply` - Modify files in place and show diff summary. Default: dry-run only.

    **## Instruction** role declaration: "You are a **manuscript cleanup specialist**."

    **STEP 1: LOAD CONTEXT**
    Following manuscript-stats.md lines 23-28 pattern:
    1. Read `.manuscript/config.json` for `title`, `author`, `work_type`
    2. Read Scriveno's installed/shared `CONSTRAINTS.json` (global `~/.scriveno/data/CONSTRAINTS.json` or project `.scriveno/data/CONSTRAINTS.json`)
    3. Prerequisite guard: if `.manuscript/drafts/` does not exist or contains no `.md` files, tell the writer to run `/scr:draft` first and **stop**.

    **STEP 2: SCAN DRAFT FILES**
    Scan all `.md` files in `.manuscript/drafts/` recursively. For each file, check line by line for:
    - Lines containing `[Fill in` - covers `[Fill in:]` and `[Fill in or delete:]`
    - Lines containing `[Delete if not applicable:]`
    - Lines containing `Alternate 1:` or `Alternate 2:`
    - Count of lines matching `^# ` (top-level H1 heading) - flag files where count > 1

    **IMPORTANT:** Do NOT flag or strip `{{VAR}}` tokens. These are unfinished writer content, not scaffold markers.

    For Alternate blocks, record the full block extent: from the `Alternate N:` line through the first of: (a) the next blank line followed by another `Alternate M:` header, or (b) a line starting with `## ` or `# `, or (c) end of file.

    For duplicate H1: identify all lines matching `^# ` in the file. The FIRST occurrence is the canonical heading. All subsequent `# ` heading lines are duplicates.

    Build a per-file list of: file path, line number, marker type and text.

    **STEP 3: APPLY OR REPORT**
    Branch on whether `--apply` was passed:

    **If no `--apply` flag (dry-run default):**
    - If no markers found across any file: output "No scaffold markers found. Your manuscript is clean."
    - For each file with markers, output the dry-run format:
      ```
      Would remove from <filename>:
        - line N: <marker text or "Alternate N: block (lines N-M)">
        - line N: <marker text>

      Summary: X bracket markers, Y Alternate blocks, Z duplicate H1 headings across F file(s).
      Run `/scr:cleanup --apply` to apply these changes.
      ```

    **If `--apply` was passed:**
    - For each file with markers: edit the file in place, removing the identified markers:
      - Bracket markers: remove the entire line containing `[Fill in...` or `[Delete if not applicable:]`
      - Alternate blocks: remove from the `Alternate N:` line through the block boundary (as defined in STEP 2)
      - Duplicate H1: remove all `# ` heading lines after the first one
    - After editing each file, output:
      ```
      Cleaned <filename>:
        - Removed X bracket markers
        - Removed Y Alternate blocks
        - Removed Z duplicate H1 headings
      ```
    - Final summary:
      ```
      Summary: X bracket markers, Y Alternate blocks, Z duplicate H1 headings removed from F file(s).
      ```
    - If nothing was found (all files already clean): output "No scaffold markers found. Manuscript already clean."
  </action>

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-30-03 | Tampering | cleanup.md `--apply` path | mitigate | Dry-run is the default; `--apply` requires explicit opt-in flag. Instruction states "show what would be removed first." Dry-run output gives writer full visibility before committing to changes. |
| T-30-04 | Information Disclosure | validate.md scan output | accept | All scanned content is local manuscript files the writer already owns. File:line output is shown only in the session; no persistent report written (per deferred section). |
