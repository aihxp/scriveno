| Boundary | Description |
|----------|-------------|
| build command -> manifest.yaml | Build commands read manifest at runtime; YAML must be well-formed to avoid parse errors |
| CONSTRAINTS.json -> command validation | JSON must remain valid after edit; malformed JSON breaks STEP 1 work-type checks in all export/build commands |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-03-01 | Tampering | data/CONSTRAINTS.json | mitigate | Verify JSON validity with `node -e "JSON.parse(...)"` after edit before committing |
| T-32-03-02 | Tampering | platform manifest YAML | mitigate | YAML is consumed by the AI agent at runtime, not eval'd; malformed YAML will cause the agent to report a read error rather than execute arbitrary code |
| T-32-03-03 | Information Disclosure | epub_notes in manifests | accept | Notes describe platform requirements, no sensitive data |
| T-32-03-04 | Denial of Service | null trim_sizes in EPUB-only manifests | mitigate | build-print.md STEP 2.5 checks formats_accepted before reading trim_sizes; EPUB-only platforms are rejected before any null access |
</threat_model>
